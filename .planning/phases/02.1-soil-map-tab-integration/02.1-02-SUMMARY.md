# Plan 02.1-02 Summary

## Outcome

Turned the soil tab into a working lazy vector overlay that renders BUEK250 polygons per Living Lab without regressing the existing landuse map path.

## Changes

- Updated `app/src/components/LLMap/index.jsx` to branch by layer type, keeping PMTiles overlays for raster tabs and adding a GeoJSON overlay path for the soil tab.
- Reworked `app/src/hooks/useGeoJSON.js` so lazy GeoJSON fetches remain URL-keyed, cached, and safe across layer or Living Lab switches without prefetching inactive soil files.
- Added deterministic soil styling keyed from `SYM_NR` / `GEN_ID`, with a separate visual treatment for water and other special-area polygons.
- Added overlay-level soil loading and error messages plus a more accurate soil legend note through `app/src/components/MapLegend.jsx` and `app/src/i18n.js`.

## Verification

- `npm run lint`
- `npm run build`
- Confirmed the production build still succeeds after enabling the soil tab and vector overlay path

## Notes

- The production build needed to be rerun outside the sandbox because Vite hit the usual Windows `spawn EPERM` restriction during config loading inside the sandbox.
