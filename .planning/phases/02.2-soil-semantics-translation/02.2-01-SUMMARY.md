# Plan 02.2-01 Summary

## Outcome

Defined and documented a dedicated BUEK250 soil semantics contract instead of growing the raw SQLite lookup inline inside `build_vector.py`.

## Changes

- Added `data-pipeline/python/soil_semantics.py` as the centralized helper for normalization, deterministic translation, special-area classification, and profile-summary assembly.
- Reworked `data-pipeline/python/build_vector.py` to load the new semantic lookup contract from the helper and apply explicit runtime fields such as `feature_kind`, `soil_label_*`, `soil_group_*`, and provenance metadata.
- Replaced the shallow `vector.lookup` declaration in `data-pipeline/sources/sources.yaml` with a richer `vector.semantics` contract that names the SQLite source tables and fallback policy.
- Documented the app-facing soil contract and fallback rules in `data-pipeline/README.md`.

## Verification

- Confirmed the rebuilt GeoJSON fixtures carry semantic fields derived from `LEGENDENEINHEIT`, `GL_EINHEIT`, `GL_BAGFlaechentyp`, `PROFIL`, and `HORIZONT`
